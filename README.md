# A Collection Catalog (Public PWA)

Public-facing Progressive Web App (PWA) catalog for **A Collection Narowal** — ladies & gents clothing retail. Customers browse products and order via WhatsApp.

**Live URL:** https://xpunjabi.github.io/a-collection-catalog/

---

## Architecture

This repository contains **only the public frontend**. All product data is generated and published by the [Head Office System](https://github.com/xpunjabi/a-collection-head-office) (a private Tauri desktop app).

```
Head Office (Tauri)                Catalog (this repo, PWA)
       │                                    │
       │ 1. User clicks "Publish"           │
       │ 2. Export catalog.json             │
       │ 3. Generate WebP images            │
       │ 4. Push via GitHub Contents API ───┼──▶ data/catalog.json
       │                                    └──▶ data/images/*.webp
       │                                    │
       │                                    │ 5. GitHub Pages auto-deploy
       │                                    │ 6. Customer browses
       │                                    │ 7. WhatsApp order link
```

**The catalog repo is treated as deployment output.** Manual edits to `data/` folder will be overwritten on next publish. Edit only `index.html`, `app.js`, `styles.css` for frontend changes.

---

## Tech Stack

- **Vanilla JS + Alpine.js** (3KB) — no build pipeline, instant load
- **Tailwind CSS** via CDN — mobile-first, no build
- **Fuse.js** (4KB) — client-side fuzzy search
- **PWA** — installable, offline caching via service worker
- **GitHub Pages** — free static hosting with HTTPS

## Local Development

No build step required. Just serve the folder with any static HTTP server:

```bash
# Option 1: Python (pre-installed on most systems)
python3 -m http.server 8080

# Option 2: Node.js (if installed)
npx serve

# Then open http://localhost:8080
```

To test with real data, replace `data/catalog.json` with output from Head Office.

## Publishing (automated by Head Office)

The Head Office Tauri app has a "Publish to Catalog" button that:

1. Exports active products to `catalog.json` (allowlist fields only)
2. Generates WebP images (400px for grid, 800px for detail)
3. Uploads via GitHub Contents API (atomic file-level updates)
4. Deletes orphan images (products removed from catalog)
5. GitHub Pages auto-deploys within ~30 seconds

**Manual edits to `data/` will be overwritten.** To customize the frontend, edit `index.html`, `app.js`, or `styles.css` — these are never touched by the publish process.

## PWA Features

- **Installable** — Add to Home Screen on Android/iOS
- **Offline** — App shell cached, products cached with stale-while-revalidate
- **Mobile-first** — Touch-optimized, 44px minimum tap targets
- **Fast** — No build pipeline, < 50KB initial load (excluding images)
- **Theme color** — Violet (#7c3aed) matches Head Office branding

## WhatsApp Integration

Every product card has a green WhatsApp button. Clicking it opens WhatsApp (app on mobile, Web on desktop) with a pre-filled message:

```

📦 Nishat 3-Piece Lawn Suit — Maroon Floral
🔖 SKU: AC-2026-001
💰 Price: Rs. 3,300
🎨 Color: Maroon
🧵 Fabric: Lawn

Is this available? Please share more details.
```

The WhatsApp number (`+92 342 0830995`) is configured in Head Office Settings and published in `catalog.json`.
